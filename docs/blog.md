## 编程初学者最容易犯的 5 个错误

**错误一：只看视频不动手。**
看教程觉得"很简单嘛"，合上视频发现自己一行都写不出来。
→ 每看 10 分钟教程，至少敲 20 分钟代码。编程是肌肉记忆。

**错误二：怕报错。**
看到红色就慌，不敢读错误信息，直接截屏问人。
→ 错误信息不是敌人，是最好的老师。它直接告诉你问题在哪行、是什么问题。

**错误三：复制 AI 代码不深究。**
能跑就行，不深究原理。
→ 可以借助 AI，但每一行都要能自己解释。解释不了的代码不是你的。

**错误四：基础没打牢就想写大项目。**
学了两周 JavaScript 就想做商城，越写越乱。
→ 先把基础语法、数据操作、事件处理搞熟，再上框架。

**错误五：自己闷头学，不跟人交流。**
→ 找同学做代码评审、写技术文章、参与开源社区。教会别人时你学到的更多。

## 从 Figma 到代码：设计师学前端怎么入门

设计师学前端有天然的优势：

1. **你已经懂了布局** — Flexbox/Grid 就是 Figma 的 Auto Layout
2. **你已经懂了层级** — z-index 就是图层面板
3. **你已经懂了组件** — Figma Components = Vue/React 组件
4. **你已经懂了交互** — 原型连线 ≈ 事件处理

<div class="code-block" markdown>

学习路线：Figma 组件思维 → HTML 结构 → CSS 布局 → JavaScript 基础 → Vue 组件开发 → "设计+工程"双修

</div>

关键心态：你不是在"转行"，是在拓展技能边界。设计能力是你的差异化优势——大部分程序员不会做设计。

## AI 时代，编程老师会被淘汰吗？

我的答案是：**不会被淘汰，但工作的内容会变。**

AI 能做的：生成模板代码、解释基本概念、调试简单错误、做代码审查。

AI 做不了的：判断学生到底卡在哪里、根据状态调整教学节奏、设计循序渐进的学习路径、激发学习动力、分享真实的项目经验和坑。

所以编程老师的工作会从"教语法"转向：教思维、教协作、教判断、教工程、教软技能。

> **AI 能把"知识的答案"给你，但老师教的是"找到答案的方法"。**

## Vue 响应式原理——试着用人话讲清楚

> 你把数据交给 Vue，Vue 帮你盯着。数据一变，页面自动跟着变。

<div class="code-block" markdown>

数据 (data) → 你关心的状态，比如 count = 0
副作用 (effect) → 数据变了要做什么，比如更新页面
追踪 (track) → Vue 偷偷记住"谁在用这个数据"

count = 5 → Proxy 发现了 → "count 改了!" → 页面自动更新

</div>

用外卖订单来类比：数据 = 你的订单状态，Proxy = 快递系统在后台盯着，副作用 = 状态一变就给你推通知，视图 = 你手机上看到的页面。

你从来不需要手动刷新——这就是"响应式"。
