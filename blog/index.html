    编程初学者最容易犯的 5 个错误

    做家教的时候发现，新手学编程遇到的问题高度重复。以下是 5 个最常见的：

    错误一：只看视频不动手。 看教程的时候觉得"很简单嘛"，合上视频发现自己一行都写不出来。 → 每看 10 分钟教程，至少敲 20 分钟代码。编程是肌肉记忆。

    错误二：怕报错。 看到红色就慌，不敢读错误信息，直接截屏问人。 → 错误信息不是敌人，是最好的老师。它直接告诉你问题在哪行、是什么问题。

    错误三：复制粘贴综合征。 从 AI 工具复制代码，能跑就行，不深究原理。 → 可以借助 AI，但每一行都要能自己解释。解释不了的代码不是你的。

    错误四：基础没打牢就想写项目。 才学了两周 JavaScript 就想做完整的商城项目，结果越写越乱。 → 先把基础语法、数据操作、事件处理这些搞熟，再上框架。

    错误五：自己闷头学，不跟人交流。 → 找同学做代码评审、写技术文章、参与开源社区。教会别人的同时你学到的更多。

    从 Figma 到代码：设计师学前端怎么入门

    我自己是从 Figma 切入前端的，这条路有天然的优势：

    1. 你已经懂了布局 — Flexbox/Grid 就是 Figma 的 Auto Layout，概念一模一样
    2. 你已经懂了层级 — z-index 就是图层面板的数字版
    3. 你已经懂了组件 — Figma Components = Vue/React 组件
    4. 你已经懂了交互 — 原型连线 ≈ 事件处理
    学习路线：
Figma 组件思维
  → HTML 结构（盒模型、语义化）
    → CSS 布局（Flexbox、Grid）
      → JavaScript 基础（变量、函数、事件）
        → Vue 组件开发
          → "设计 + 工程"双修

    关键是心态：你不是在"转行"，而是在拓展技能边界。设计能力是你的差异化优势——大部分程序员不会做设计。

    AI 时代，编程老师会被淘汰吗？

    我的答案是：不会被淘汰，但工作的内容会变得不一样。

    AI 能做的事：生成模板代码、解释基本概念、调试简单错误、做代码审查。这些以前是老师工作的一部分。

    AI 做不了的事：判断学生到底卡在哪里、根据状态调整教学节奏、设计循序渐进的学习路径、激发学习动力、分享真实的项目坑和经验。

    所以编程老师的工作会从"教语法"转向：

    1. 教思维 — 计算思维、问题拆解、方案设计
    2. 教协作 — 怎么和 AI 高效配合，而不是依赖
    3. 教判断 — 怎么分辨好代码和烂代码
    4. 教工程 — 实际项目的工作流、工具链、团队协作
    5. 教软技能 — 沟通、文档、代码评审

    一句话总结：AI 能把"知识的答案"给你，但老师教的是"找到答案的方法"。

    Vue 响应式原理——试着用人话讲清楚

    你把数据交给 Vue，Vue 帮你盯着。数据一变，页面自动跟着变。

    三个需要知道的词：

    数据 (data)     →  你关心的状态，比如 "count = 0"
副作用 (effect) →  数据变了你要做什么，比如 "更新页面上的数字"
追踪 (track)    →  Vue 偷偷记住"谁在用这个数据"

    流程是这样的：

    count = 5           ← 你改了数据
    ↓
Proxy 发现了         ← Vue 3 用 Proxy 暗中监听
    ↓
"count 改了!"        ← 通知所有用到 count 的地方
    ↓
页面自动更新          ← 你不用管，Vue 帮你搞定

    用外卖订单来类比：

    • 数据 = 你的订单状态
    • Proxy = 快递系统在后台盯着
    • 副作用 = 状态一变就给你推通知
    • 视图 = 你手机上看到的页面

    你从来不需要手动刷新——这就是"响应式"。
